## 项目简介

`copyright-crawler` 是一个基于 Go 编写的自动化脚本，用于登录中国版权保护中心并获取省、市、区（县）三级地区数据。当前实现并不是单纯依赖页面 DOM 抓取，而是组合了以下几步：

- 使用 [`chromedp`](https://github.com/chromedp/chromedp) 驱动浏览器完成登录流程。
- 监听浏览器网络请求，提取 `token`、`Authorization` 和相关请求头。
- 从浏览器上下文中获取 Cookie，并复用到后续 HTTP 请求。
- 直接调用网关地区接口，按省 -> 市 -> 区县的层级拉取数据。
- 将原始接口结果转换为结构化 JSON，写入项目根目录的 `output.json`，并在终端输出摘要。

## 当前能力

- 自动打开可视化浏览器并进入登录页。
- 自动填充用户名、密码并点击登录。
- 保留人工处理验证码的窗口，适合滑块或图形校验场景。
- 在登录后监听页面请求，优先从真实请求头中提取鉴权信息。
- 当请求头未拿到 token 时，回退到 `localStorage` 中读取。
- 复用登录态中的请求头、`authorization_key` 和 Cookie 调用地区接口。
- 输出整理后的三级地区 JSON 数据到 `output.json` 文件。
- 在页面结构变化时导出调试 HTML，便于后续排查。

## 快速开始

### 1. 配置账号

编辑 [`main.go`](main.go)，替换登录凭证：

```go
loginURL := "https://register.ccopyright.com.cn/login.html"
username := "YOUR_USERNAME"
password := "YOUR_PASSWORD"
```

当前代码中账号密码仍为占位符，必须手动替换后才能运行。

### 2. 安装依赖

```bash
go mod tidy
```

### 3. 启动程序

```bash
go run main.go
```

运行完成后，程序会在项目根目录生成或覆盖 `output.json`，终端只打印结果文件路径、数据条数和完成提示。

## 运行流程

1. 程序以非无头模式启动浏览器并打开登录页。
2. 自动填写账号密码并点击“立即登录”。
3. 如果出现验证码，需要在浏览器中手动完成验证。
4. 登录成功后，程序等待页面跳转离开 `login.html`。
5. 程序监听网关请求，提取鉴权头和 token。
6. 程序尝试执行一段页面内 UI 操作，用于进入业务流程并触发相关请求。
7. 程序读取浏览器 Cookie，并直接请求地区接口：
   - 省份接口：`/userServer/area/province/001`
   - 城市/区县接口：`/userServer/area/city/{id}/1`
8. 程序将接口返回结果转换为统一结构，写入项目根目录的 `output.json`。

## 输出示例

写入 `output.json` 的 JSON 结构大致如下：

```json
[
  {
    "Province": "北京市",
    "Cities": [
      {
        "City": "北京市",
        "Districts": [
          {
            "District": "东城区"
          }
        ]
      }
    ]
  }
]
```

说明：

- 最终输出中保留 `Province`、`City`、`District` 字段。
- 对应的编码字段在内部处理中存在，但默认不输出到最终 JSON。
- 终端默认不打印完整 JSON，只输出结果文件路径、顶层数量和完成提示。

## 调试说明

如果页面结构发生变化，当前代码会在部分关键步骤失败时导出页面 HTML：

- `debug_page.html`：等待“软件登记”入口超时时生成。
- `debug_agent_page.html`：等待“请输入软件全称”输入框失败时生成。

这两个文件不会默认提交到仓库，但在本地调试页面结构变更时很有用。

## 已知限制

- 验证码无法全自动处理，仍需要人工介入。
- `main.go` 中存在若干与当前业务页面强绑定的 XPath 和文本匹配，页面改版后容易失效。
- 当前“软件名称”和“版本号”步骤写入的是固定测试值：
  - 软件全称：`测试测试`
  - 版本号：`1.0.0`
- 项目暂未拆分配置文件、命令行参数或独立测试用例，整体仍属于单文件脚本形态。
